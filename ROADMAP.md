# Roadmap

## Current focus

PanePilot is optimizing for one thing: making terminal-native coding agents
stable enough to use as long-running workspaces on real machines.

## Near-term

- more runtime-specific profiles and validation notes
- stronger unhealthy-state detection
- better startup readiness heuristics
- cleaner session listing and filtering
- more portable demo and installation guidance

## Mid-term

- better handling for multiple named PanePilot configs
- optional command to scaffold config from a runtime template
- richer logging and event breadcrumbs
- tighter compatibility guidance for common agent CLIs

## Explicitly not the goal

- replacing tmux
- becoming a hosted orchestration platform
- becoming a multi-agent workflow engine
- abstracting away every runtime difference
