# Publish Checklist

## Before making the repo public

- Remove any remaining personal paths or machine-specific assumptions.
- Confirm `config/panepilot.env` is not tracked.
- Verify the default examples work with `codex` or `claude`.
- Add one screenshot or short terminal GIF to the README.
- Make the GitHub description match the first README line.
- Decide whether to keep the repository name as `panepilot`.

## Before launch posts

- Prepare a 30 to 45 second terminal demo.
- Write one concrete use case:
  "I keep Codex running in tmux and inject review tasks throughout the day."
- Write a short comparison line:
  "Not an orchestrator. A durable workspace layer."
- Tag the right audiences:
  tmux, terminal tooling, coding agents, Claude Code, Codex, OpenCode

## After launch

- Watch for install friction in the first five user comments.
- Tighten the quick start before adding new features.
- Add one more example config for a second agent runtime.
